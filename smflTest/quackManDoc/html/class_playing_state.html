Detailed Description

Playing State Class This class inherits from the GameState Class. This game state is where everything is implemented. The game is played in this state. It takes care of loading the levels, calculating scores, checking for Player VS Ghost Collisions, picking up pellets, and rendering the HUD.

◆ update()

void PlayingState::update ( sf::Time  a_delta)
virtual

This function does two major things: Score Counting and Collision Checking Collision Checking: It checks if the player collided with "strong" ghosts, "weak" ghosts, dots, super dots, or a bonus. If the player collided with a "strong" ghost then the player loses a life and all the character entities resets to its original position. If the player collides with anything else it earns scores Score Counting: The player earns score depending on what it collides with or "eats". Dots are 5 pts, SuperDots are 25 pts, "Weak" Ghosts are 100 pts, and the bonus, which shows up when there are only 150 dots left in the maze, is 500 pts

a_deltathe current time
